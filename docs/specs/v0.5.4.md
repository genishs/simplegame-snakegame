# v0.5.4 Spec

## Goal

4방향 절대 방향 입력을 좌/우 상대 회전 모델(← → / A D)로 전면 교체하고, 모바일 세로화면에 하단 3버튼 + viewport-fit 반응형 레이아웃을 추가하여 Issue #4를 닫는다.

## User Stories

- 플레이어로서, ← / A 로 현재 이동 방향 기준 좌로 90° 회전하고 → / D 로 우로 90° 회전하고 싶다. 그래서 절대 방향이 아닌 내 뱀의 시점에서 직관적으로 조작할 수 있다.
- 모바일 플레이어로서, 화면 하단의 큼직한 좌/우 회전 버튼을 엄지로 눌러 뱀을 조종하고 싶다. 그래서 키보드 없이도 쾌적하게 플레이할 수 있다.
- 플레이어로서, 캔버스 왼쪽 절반을 클릭/터치하면 좌 회전, 오른쪽 절반을 클릭/터치하면 우 회전하고 싶다. 그래서 마우스나 손가락으로도 빠르게 방향을 바꿀 수 있다.
- 모바일 플레이어로서, 게임 시작 전 캔버스에 옅은 좌/우 터치존 힌트가 보이고 첫 입력 시 사라지길 원한다. 그래서 조작법을 별도 설명 없이 바로 이해할 수 있다.
- 플레이어로서, 어떤 화면 크기에서도 캔버스가 뷰포트를 벗어나지 않고 스크롤 없이 플레이하고 싶다. 그래서 기기에 상관없이 쾌적한 플레이 환경이 유지된다.

## Acceptance Criteria

### Set A — 키보드 입력 모델 교체

- [ ] `←` 키 또는 `A` / `a` 키 입력 시 현재 이동 방향 기준 좌 90° 회전(반시계)이 적용된다.
- [ ] `→` 키 또는 `D` / `d` 키 입력 시 현재 이동 방향 기준 우 90° 회전(시계)이 적용된다.
- [ ] `↑` / `↓` / `W` / `w` / `S` / `s` 키는 무시된다. `preventDefault`를 호출하지 않아 페이지 스크롤이 그대로 동작한다.
- [ ] 기존 `dirFromKey()` 함수는 `rotateLeft()` / `rotateRight()` 헬퍼(또는 동등한 인라인 로직)로 교체되며, ↑ ↓ W S 분기는 제거된다.
- [ ] `setDirection(dx, dy)` 함수 시그니처는 보존된다. 내부 180° 가드(`dx === -dir.x && dy === -dir.y` 반환 조건)는 제거되고, 90° 회전 결과를 `nextDir`에 큐잉하는 로직으로 재작성된다.
- [ ] `Space` 동작은 변경 없다(start / pause / restart 1:1 매핑). `Esc` 동작은 변경 없다(튜토리얼 skip).

### Set B — 회전 수학

- [ ] 현재 `dir`이 `(1, 0)` (오른쪽) 일 때: 좌 회전 → `(0, -1)`, 우 회전 → `(0, 1)`.
- [ ] 현재 `dir`이 `(0, 1)` (아래) 일 때: 좌 회전 → `(1, 0)`, 우 회전 → `(-1, 0)`.
- [ ] 현재 `dir`이 `(-1, 0)` (왼쪽) 일 때: 좌 회전 → `(0, 1)`, 우 회전 → `(0, -1)`.
- [ ] 현재 `dir`이 `(0, -1)` (위) 일 때: 좌 회전 → `(-1, 0)`, 우 회전 → `(1, 0)`.
- [ ] 한 tick 안에 좌/우 입력이 두 번 이상 발생하면 마지막 입력만 `nextDir`에 보존된다(기존 nextDir 큐잉 동작 유지).
- [ ] 180° 가드가 제거되어 있으므로 U턴 입력 자체는 큐잉 시점에 차단되지 않는다. 회전 수학상 두 번의 90° 회전 없이는 U턴이 발생하지 않으므로 별도 가드 불필요.

### Set C — 모바일 매체쿼리 + 레이아웃

- [ ] `@media (max-aspect-ratio: 1/1) and (hover: none) and (pointer: coarse)` 조건 진입 시 하단 컨트롤 영역이 표시된다.
- [ ] 위 조건 외(데스크탑, 가로화면 등)에서는 하단 컨트롤 영역이 숨겨진다(`display: none` 또는 동등).
- [ ] 캔버스 CSS `width` / `height`는 `min(100vw - 32px, 100vh - var(--mobile-controls-reserve), 400px)` 공식(또는 동등 clamp)으로 viewport-fit 된다. 픽셀 버퍼 `width="400" height="400"` 속성은 변경되지 않는다.
- [ ] `--mobile-controls-reserve`: 모바일 매체쿼리 안에서 `var(--mobile-controls-height) + var(--canvas-mobile-gap) + 32px`으로 설정, 그 외에서 `0px`.
- [ ] 캔버스 `aspect-ratio: 1 / 1` CSS 규칙이 적용된다.
- [ ] 어떤 화면 크기에서도 캔버스가 뷰포트 밖으로 나가지 않으며 페이지 스크롤이 발생하지 않는다.
- [ ] `safe-area-inset-bottom`이 하단 컨트롤 영역 하단 여백에 적용된다(`padding-bottom: env(safe-area-inset-bottom, 0px)` 또는 동등).

### Set D — 모바일 하단 버튼 3개

- [ ] 하단 컨트롤 영역에 버튼 3개가 한 행으로 배치된다: `[좌 회전] [aux] [우 회전]` 순서, `justify-content: center`.
- [ ] 좌 회전 버튼: 크기 `72px × 72px` (`--mobile-rotate-btn-size`), `aria-label="Rotate left"`, 반시계방향 원형 화살표 inline SVG 아이콘.
- [ ] 우 회전 버튼: 크기 `72px × 72px` (`--mobile-rotate-btn-size`), `aria-label="Rotate right"`, 시계방향 원형 화살표 inline SVG 아이콘.
- [ ] aux 버튼(가운데): 크기 `52px × 52px` (`--mobile-aux-btn-size`), `aria-label`은 게임 상태에 따라 "Start" / "Pause" / "Resume" 중 하나로 동적 갱신. inline SVG 아이콘은 IDLE/PAUSED/GAME_OVER일 때 재생 삼각형, PLAYING일 때 일시정지 두 막대.
- [ ] 아이콘은 inline SVG만 사용한다. 유니코드 문자(↺ ↻ ▶ ⏸)는 사용하지 않는다.
- [ ] SVG `viewBox="0 0 24 24"`, `stroke-width="2.5"`, `stroke-linecap="round"`, `stroke-linejoin="round"`, `currentColor` 사용.
- [ ] `<button>` 요소이며, `border-radius: var(--mobile-btn-radius)` (`18px`), 배경 `var(--mobile-btn-bg-idle)`, 그림자 `var(--mobile-btn-shadow)` (2단 shadow) 적용.
- [ ] `:active` 시 shadow offset이 `0 1px 0 ...`으로 축소(눌리는 느낌).
- [ ] 버튼 간격 `gap: var(--mobile-btn-gap)` (`24px`).
- [ ] 좌 회전 버튼 클릭/터치 → `rotateLeft()` (또는 동등 동작), 우 회전 버튼 → `rotateRight()`. state가 PLAYING이면 즉시 적용, BLOCKED이면 tryUnblock 로직 경유.
- [ ] aux 버튼 클릭/터치 → Space 키와 동일한 분기(`state === PLAYING` → pause, 그 외 → start).
- [ ] 버튼 라벨 텍스트(버튼 내 표시 문자)는 없다. 아이콘만 표시.

### Set E — 터치존 힌트

- [ ] 터치존 힌트는 `STATE.IDLE` 및 `STATE.READY`에서만 표시된다.
- [ ] 힌트는 캔버스의 최상단 레이어(draw 함수 마지막 단계)로 그려진다. 캔버스 좌측 절반: `rgba(201, 165, 116, 0.08)` 직사각형, 우측 절반: 동일.
- [ ] 가운데 구분선은 없다.
- [ ] 페이드인: IDLE 화면 렌더 후 300ms 지연, 이후 400ms ease-out으로 alpha 0 → 0.08.
- [ ] 페이드아웃: 첫 입력(keydown 중 좌/우 회전 키, pointerdown on canvas, 하단 버튼 press) 발생 시 200ms ease-in으로 alpha 0.08 → 0 후 숨김. 이후 같은 게임 세션에서 다시 표시되지 않는다.
- [ ] 힌트 표시 여부는 별도 플래그(`hintDismissed`)로 관리한다. 게임 리셋 시 플래그도 초기화한다.

### Set F — 포인터 통합 (캔버스 클릭/터치)

- [ ] 캔버스에 `pointerdown` 이벤트 핸들러 하나를 등록한다. 마우스, 터치, 펜 구분 없이 동일 처리.
- [ ] 캔버스 픽셀 좌표 변환: `rect = canvas.getBoundingClientRect()`로 캔버스 CSS 크기를 구한 뒤 `pixelX = (e.clientX - rect.left) / rect.width * 400`으로 픽셀 버퍼 좌표로 환산한다.
- [ ] `pixelX < 200` → 좌 회전, `pixelX >= 200` → 우 회전.
- [ ] `state === PLAYING` → `setDirection` 경유 회전 적용, `state === BLOCKED` → `tryUnblock` 경유.
- [ ] `state === IDLE` / `STATE.READY` / `STATE.PAUSED` / `STATE.GAME_OVER` 에서의 캔버스 pointerdown은 좌/우 회전을 큐잉하되 게임을 자동 시작하지 않는다(게임 시작은 Space / aux 버튼 전용).
- [ ] 캔버스 외 영역(HUD, 컨트롤 스트립, 페이지 여백) pointerdown은 이 핸들러가 처리하지 않는다.

#### Edge case — IDLE 상태에서의 회전 입력

IDLE 상태(게임 시작 전)에서 좌/우 회전 입력(키보드, 캔버스 클릭, 하단 버튼)이 들어오면 무시한다. 게임 시작은 Space / aux 버튼만으로 트리거된다. 첫 진행 방향은 항상 오른쪽 `(1, 0)`으로 고정된다.

#### Edge case — BLOCKED 상태(튜토리얼)에서의 회전 입력

BLOCKED 상태에서 좌/우 회전 입력이 들어오면, 회전 결과 방향(`dir`에 90° 회전 적용한 값)을 계산한 뒤 `isSafeDir(rotated)` 검사를 수행한다. `isSafeDir` 통과 시 `tryUnblock(rotated.x, rotated.y)` 호출로 BLOCKED 해제 후 게임 재개. 통과 실패 시 BLOCKED 유지, 입력 무시.

### Set G — 컨트롤 안내 갱신

- [ ] `index.html`의 `.hint` 텍스트가 다음으로 교체된다: `← → / A D — Turn left/right &nbsp;·&nbsp; Space — Start/Pause/Restart &nbsp;·&nbsp; Esc — Skip tutorial`
- [ ] `index.html` 버전 표기가 `v0.5.4`로 갱신된다.
- [ ] `README.md` Controls 섹션(또는 표)이 새 입력 모델에 맞게 갱신된다: ↑ ↓ / W S 행 제거, ← → / A D 행을 "Turn left / Turn right"로 기재.

### Set H — 버전 및 메타 파일

- [ ] `VERSION` 파일이 `0.5.4`로 갱신된다.
- [ ] `HISTORY.md` 최상단에 v0.5.4 항목이 추가된다. PR 본문에 `Closes #4` 포함.
- [ ] `README.md` Roadmap 표에 v0.5.4 행이 추가된다.

## Non-functional Notes

### 페이로드

- 총 예상 추가량: CSS ~1.2KB + SVG ~0.4KB + JS ~1.5KB = ~3.1KB. 현재 ~21KB → 예상 ~24KB, 50KB 상한 이내 (STYLE.md v0.5.4 Payload check 참조).

### 성능 (60fps)

- 추가 연산: 힌트 알파 보간 1회 (프레임당 수치 계산 1~2 op), 포인터 좌표 변환 1회 (이벤트 당), 버튼 이벤트 핸들러 (이벤트 당). 렌더 루프 부담 없음.
- 터치존 힌트는 프레임당 직사각형 fillRect 2회 추가 — 미미.

### 접근성

- 하단 버튼은 `<button>` 요소 + `aria-label` (스크린리더 접근 가능).
- 버튼 크기: 좌/우 회전 72px, aux 52px — 둘 다 44px 접근성 최소치 이상.
- OS 레벨 햅틱은 `<button>` `:active` 상태에서 일부 기기에서 무료 제공 (별도 코드 없음).
- 키보드 전용 사용자: Space / Esc / ← → / A D 모두 키보드로 접근 가능.

### 모바일 고려사항

- 단일 매체쿼리 `(max-aspect-ratio: 1/1) and (hover: none) and (pointer: coarse)` 를 유일한 분기점으로 사용. `max-width` 브레이크포인트 추가 금지.
- `env(safe-area-inset-bottom)` 적용으로 iOS 홈 인디케이터 / Android 제스처 바 침범 방지.
- 가로화면: 버튼 미노출, 캔버스 viewport 중앙 fit (CSS `height` 기준 90vh까지).
- `touch-action: manipulation` 을 캔버스 및 버튼에 적용하여 300ms tap delay 제거 권장.

## Decisions

- **(A) IDLE 상태 회전 입력 무시** — 회전 입력이 게임 시작을 트리거하면 실수로 게임이 시작될 수 있고, 첫 방향이 사용자 의도와 다를 수 있다. Space / aux 버튼을 명확한 시작 액션으로 단일화하여 인지 부하를 낮춘다.
- **(B) BLOCKED tryUnblock — 회전 결과에 isSafeDir 적용** — 기존 `tryUnblock(dx, dy)`는 절대 방향을 받는다. 회전 모델에서는 입력이 "left" / "right"이므로 현재 `dir` 기준 90° 회전한 결과 벡터를 계산한 뒤 `isSafeDir`에 전달한다. 로직 재사용을 최대화하면서 튜토리얼 hint 동작을 보존한다.
- **(C) pointerdown 좌표 환산에 rect 비율 사용** — 캔버스 CSS 크기가 viewport-fit으로 동적으로 변하므로, `getBoundingClientRect().width`로 현재 표시 크기를 구한 뒤 `/ rect.width * 400`으로 픽셀 버퍼 좌표로 변환한다. 이 방법은 CSS transform/scale이 없는 한 항상 정확하다.
- **(D) 180° 가드 제거** — 회전 모델에서는 단일 좌/우 입력으로 U턴이 수학적으로 불가능하다. 가드는 불필요한 dead code가 되므로 제거하여 코드 단순화.
- **(E) 힌트 플래그 리셋 포함** — `hintDismissed` 플래그를 게임 리셋 시 초기화하면, 새 게임을 시작하는 신규 플레이어에게 힌트가 다시 표시된다. 이미 익숙한 플레이어는 첫 입력 즉시 제거하므로 방해되지 않는다.
