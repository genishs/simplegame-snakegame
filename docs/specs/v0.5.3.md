# v0.5.3 Spec

## Goal

사과를 먹으면 시각 전용 '소화 덩어리(bulge)' 객체가 뱀 머리에서 생성돼 body polyline을 따라 꼬리까지 흘러간 뒤 200ms 페이드아웃된다. 게임 로직·grow·점수는 v0.5.2와 동일하게 즉시 처리된다.

## User Stories

- 플레이어로서, 사과를 먹었을 때 몸통 안에서 덩어리가 꼬리 쪽으로 천천히 흘러가는 것을 보고 싶다. 그래서 뱀이 음식을 소화하는 것처럼 느껴진다.
- 플레이어로서, 연속으로 사과를 빠르게 먹어도 각 덩어리가 독립적으로 흘러가는 것을 보고 싶다. 그래서 여러 번의 먹기 동작이 각각 시각적으로 확인된다.
- 플레이어로서, 덩어리가 꼬리에 도달하면 자연스럽게 사라지는 것을 보고 싶다. 그래서 소화 완료가 부드럽게 전달된다.

## Acceptance Criteria

### Set A — Spawn 및 배열 관리

- [ ] 사과를 먹는 순간 새 bulge 객체가 `bulges` 배열에 push된다. `bulge.progress = 0`, `bulge.position = 머리 cell` 로 초기화된다.
- [ ] `bulges` 배열 최대 용량은 8개이다. 8개 초과 시 가장 오래된(index 0) bulge를 즉시 제거하고 새 bulge를 push한다 (게임 로직 영향 없음).
- [ ] 페이드 진행 중인 bulge가 있더라도 새 bulge spawn은 정상적으로 허용된다. 페이드 중 bulge와 이동 중 bulge는 배열 안에서 독립적으로 추적된다.
- [ ] snake 길이가 2 이하일 때 bulge가 spawn되면, spawn은 정상적으로 수행되되 즉시 fade 단계에 진입한다 (travel 구간이 없거나 매우 짧으므로 빠르게 보이고 사라짐).

### Set B — 이동 및 위치 계산

- [ ] 매 프레임 `state === PLAYING`일 때만 `bulge.progress += bulgeFlowSpeed × deltaSeconds`가 갱신된다. BLOCKED 상태(일시정지 포함)에서는 progress가 멈춘다.
- [ ] `bulgeFlowSpeed = 2.0` cells/sec 이다.
- [ ] bulge 위치 계산: `progress`의 정수부 = segment 인덱스, 소수부 = 두 인접 segment 사이 보간 비율이다.
  - 직선 구간: 두 segment center 사이 선형 보간(lerp).
  - 회전 cell(corner): body 드로잉에서 사용하는 것과 동일한 `quadraticCurveTo` 제어점을 재사용하여 이차 베지어를 소수부 `t`로 평가한다. bulge가 body stroke 바깥으로 벗어나지 않는다.
- [ ] bulge 진행 방향 tangent는 직선 구간에서는 segment 방향 벡터, corner 구간에서는 해당 `t`에서의 이차 베지어 도함수로 계산된다. ellipse의 장축 회전에 사용된다.
- [ ] `bulge.progress >= snake.length - 1` 이 되는 순간 fade 단계에 진입한다. progress 초과분은 버린다.

### Set C — 스케일 보간

- [ ] bulge 크기: `scale = lerp(bulgeMaxScale, bulgeMinScale, progressFraction)` 로 계산한다.
  - `progressFraction = progress / (snake.length - 1)` (spawn 시점 기준 body 길이로 snapshot, 이후 snake 성장에 의해 retroactively 변경되지 않음).
  - `bulgeMaxScale = 0.80`, `bulgeMinScale = 0.60`.
- [ ] 페이드 단계에서 scale은 `bulgeMinScale`(0.60)로 고정된다. scale이 fade 도중 추가로 줄어들지 않는다.

### Set D — 페이드 아웃

- [ ] 페이드 단계 지속 시간은 `bulgeFadeMs = 200ms`이다.
- [ ] 페이드 중 alpha는 `1.0 → 0.0`으로 선형 감소한다. scale은 `bulgeMinScale`로 고정.
- [ ] `bulgeFadeMs` 경과 후 해당 bulge를 `bulges` 배열에서 즉시 제거한다.

### Set E — 드로잉

- [ ] bulge는 `ctx.ellipse()`로 그린다.
  - 채우기 색: `bulgeFill = #d76461`.
  - 알파: 이동 중 1.0, 페이드 중 선형 감소.
  - 가로(장축, 진행 방향) : 세로(단축, 수직) = 1.15 : 1.0.
  - 단축 width = `--body-thickness × 0.95` 를 캡으로 사용한다. 장축은 단축 × 1.15.
  - 회전: 현재 진행 방향 tangent 벡터로 ellipse를 회전시킨다.
- [ ] bulge는 body stroke 위, head ellipse 아래 순서로 그린다.
  - 드로잉 순서: (1) body shadow stroke, (2) body main stroke, (3) bulge ellipses, (4) head ellipse + face, (5) apple + HUD.

### Set F — 상태 정리

- [ ] 게임오버 시 `bulges` 배열이 비워진다 (`reset()` 또는 `gameOver()` 내부).
- [ ] Tutorial Esc(skip) 시 `bulges` 배열이 비워진다.

### Set G — Carryover cleanup

- [ ] `computeSquash()` 함수 내 미사용 지역 변수 `eased`가 제거된다 (single-line cleanup).

### 버전 관리

- [ ] `VERSION` 파일이 `0.5.3`으로 갱신된다.
- [ ] 페이지(`index.html`) 표시 버전이 `v0.5.3`으로 갱신된다.
- [ ] `HISTORY.md`에 v0.5.3 항목이 추가된다. 마지막 줄에 `Carryover: computeSquash() unused eased variable removed.` 를 기록한다.

## Non-functional Notes

### 페이로드
- 전체 페이로드 50KB 이하를 유지한다. v0.5.3 추가 코드는 +1.5~2.0KB 예상 (STYLE.md v0.5.3 Performance budget 참조). 현재 ~18.9KB → 예상 ~20.5~21.0KB.

### 성능 (60fps)
- 최악 조건(snake 60 cells, bulge 8개): 프레임당 추가 arc-length walk ~80 arithmetic ops + ellipse draw call 8회. v0.5.2 body+head 비용 이후에도 16.7ms 예산 내 안전 (STYLE.md v0.5.3 Performance budget 참조).
- bulges 배열은 최대 8개로 상한이 고정되어 있으므로 GC 부담 없음. 가능하면 배열 push/splice를 inline으로 처리하여 중간 할당 회피를 권장.
- PR에서 DevTools Performance 패널로 60-segment + 8-bulge 상태의 frame time이 16.7ms 미만임을 확인한다.

### 접근성 / 모바일
- 순수 시각 효과이며 키보드/WASD 조작은 기존 그대로 유지된다. 터치 조작은 이번 버전 범위 밖이다.

## Decisions

- **(A) snake 길이 2 이하에서 즉시 fade 진입** — travel 구간이 실질적으로 없는 edge case에서 spawn을 막으면 "먹었는데 아무 반응이 없다"는 느낌을 준다. spawn 후 즉시 fade 단계로 전환하면 시각적 피드백은 유지하면서 travel 로직을 우회할 수 있다.
- **(B) BLOCKED/일시정지 시 멈춤** — bulge progress는 `deltaSec` 기반이므로 별도 멈춤 처리가 없으면 state와 무관하게 진행된다. `state === PLAYING` 조건으로 guard하면 일시정지·game-over 화면에서 bulge가 계속 움직이는 시각적 이질감을 제거할 수 있다.
- **(C) 페이드 중 scale 고정** — fade 중 scale을 추가로 줄이면 "공중으로 사라지는" 느낌이 든다. alpha-only 페이드는 "뱀에게 흡수된다"는 소화 완료 느낌을 더 잘 전달한다 (STYLE.md Fade behavior 참조).
- **(D) 드로잉 순서 — body 위, head 아래** — 새 bulge가 머리 위치에서 spawn될 때 head ellipse 아래에서 나타나도록 하여 "머리 안에서 밀려 나오는" 느낌을 준다. apple이나 HUD를 가리지 않는다.
- **(E) computeSquash() eased 변수 cleanup** — v0.5.2에서 이미 미사용인 변수이며 v0.5.3 diff에 carryover로 포함하여 dead code를 누적시키지 않는다.
