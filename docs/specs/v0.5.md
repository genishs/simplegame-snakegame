# v0.5 Spec

## Goal
튜토리얼이 "처음 배우는 사람을 위한 시간"이 되도록 한다. 속도를 추가로 늦추고, 충돌이 발생할 상황에서 게임오버 대신 안내를 띄우고 플레이어가 안전한 방향을 선택할 때까지 기다린다.

## User Stories
- 첫 진입 플레이어로서, 튜토리얼이 너무 빨라서 따라잡지 못하는 일을 피하고 싶다.
- 첫 진입 플레이어로서, 잠깐 실수해도 처음부터 다시 시작하지 않고 그 자리에서 다시 시도하고 싶다.
- 기존 플레이어로서, Stage 1 이후의 게임은 여전히 충돌 시 끝나길 원한다.

## Acceptance Criteria
- [ ] 튜토리얼 틱 속도가 350ms 이다 (v0.4의 200ms 대비 75% 더 느림)
- [ ] 튜토리얼 진행 중, 다음 이동이 벽 밖으로 나가는 경우 — 이동을 적용하지 않고 BLOCKED 상태로 진입한다
- [ ] 튜토리얼 진행 중, 다음 이동이 자기 몸과 겹치는 경우 — 이동을 적용하지 않고 BLOCKED 상태로 진입한다
- [ ] BLOCKED 상태에서는 화면에 "잠깐! 다른 방향을 눌러주세요" 안내가 표시된다
- [ ] BLOCKED 상태에서는 틱이 진행되지 않는다 (뱀은 그 자리에 머무른다)
- [ ] BLOCKED 상태에서 플레이어가 *안전한* 방향키를 누르면 즉시 PLAYING으로 복귀하고 안내가 사라진다
- [ ] BLOCKED 상태에서 *안전하지 않은* 방향키(현재 방향, 또는 여전히 충돌하는 방향)를 누르면 BLOCKED 유지
- [ ] BLOCKED 상태에서 Space를 눌러도 일시정지로 전환되지 않는다 (BLOCKED는 자체 안내 상태)
- [ ] 일반 스테이지(Stage 1 이상)에서는 v0.4와 동일하게 충돌 시 게임오버 (행동 변경 없음)
- [ ] 튜토리얼 클리어 흐름(사과 3개 → 안내 → Stage 1)은 v0.4와 동일
- [ ] 튜토리얼 진행 중 `Esc` 키를 누르면 즉시 Stage 1으로 전환된다 (점수 유지)
- [ ] Esc로 스킵 시 STAGE_CLEAR 안내는 표시하지 않고 즉시 전환
- [ ] 일반 스테이지에서는 `Esc` 입력이 무시된다
- [ ] 컨트롤 안내에 "Esc — Skip tutorial" 문구가 추가된다
- [ ] `VERSION` = `0.5`, 페이지 표시 = `v0.5`, `HISTORY.md`에 v0.5 항목

## Out of Scope
- 일반 스테이지의 게임오버 동작 변경
- 튜토리얼 사과 개수 변경
- 새로운 시각 토큰 / 디자인 변경
- 튜토리얼 외 다른 스테이지에 noFailOnHit 적용

## Non-functional notes
- 페이로드 예산 50KB 유지
- BLOCKED 검사는 `tick()` 안에서 동기적으로 수행 (별도 setTimeout 없음)
- "안전한 방향" 판단: 현재 머리에서 그 방향으로 이동했을 때 벽 밖이 아니고, 자기 몸(꼬리 제외)과 겹치지 않으면 안전
- 키 입력 매핑 변경 없음 (방향키/WASD/Space)

## Open Questions
- BLOCKED 상태에서 사과를 먹는 시나리오는 불가능 (이동이 적용되지 않으므로). 명세 외 동작 없음을 확인.
- 안내 카피 "잠깐! 다른 방향을 눌러주세요" — 다국어 화 검토는 별도 버전.
