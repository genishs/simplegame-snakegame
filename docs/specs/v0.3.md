# v0.3 Spec

## Goal
캔버스에 "아기자기한 뱀이 사과를 먹는" 비주얼을 입힌다. 게임플레이 규칙은 그대로 두고 보이는 모양만 바꾼다.

## User Stories
- 플레이어로서, 화면에 표시된 음식이 "사과"라는 것을 한눈에 알아보고 싶다. 그래야 게임이 친근하게 느껴진다.
- 플레이어로서, 뱀이 살아 있는 캐릭터처럼 보이길 원한다 (얼굴/눈). 그래야 정 가는 게임이 된다.
- 플레이어로서, 사과를 먹는 순간이 미세하게라도 시각적으로 반응하길 원한다. 그래야 보상감이 생긴다.

## Acceptance Criteria
- [ ] 음식 자리에 빨간색 사각형 대신 사과(빨간 본체 + 초록 잎 + 흰색 하이라이트)가 그려진다
- [ ] 뱀 머리 셀에 두 개의 눈(흰자+검은자)이 진행 방향에 맞춰 그려진다
- [ ] 뱀의 모든 셀이 둥근 모서리로 렌더링된다 (반지름 ≥ 4px)
- [ ] 정지된 사과는 약 1.2초 주기로 미세하게 위아래 wobble (±1.5px) 한다
- [ ] 사과를 먹은 직후 0.15초 동안 머리 셀이 살짝 확대(110%)되었다가 원복된다
- [ ] 캔버스 배경에 옅은 그리드 라인(투명도 낮은 한 가지 색)이 깔린다
- [ ] 게임 시작/일시정지/게임오버/재시작 흐름은 v0.2와 동일하게 작동한다
- [ ] 60fps 환경에서 끊김 없이 부드럽게 렌더링된다 (틱과 분리된 애니메이션 프레임 사용)
- [ ] `VERSION` 파일이 `0.3`, `index.html`의 표시 버전이 `v0.3`이다
- [ ] `HISTORY.md`에 v0.3 항목이 추가되어 있다
- [ ] `docs/design/STYLE.md`가 존재하며 본 PR에서 사용한 모든 색상/반지름/모션 값이 토큰으로 정의되어 있다

## Out of Scope
- 스테이지/난이도 진행
- 효과음/배경음
- 모바일 터치 입력
- 새 키 바인딩
- 사과 외의 다른 음식 종류

## Non-functional notes
- 페이로드 예산: 총 파일 크기 50KB 이하 유지 (현재 ~6KB)
- 빌드 도구 없음 — vanilla JS, inline SVG 또는 캔버스 프리미티브만
- 접근성: 키보드-only 조작 변경 없음
- 성능: `requestAnimationFrame` 도입 시 게임 로직 틱(110ms)과 렌더 프레임을 분리할 것

## Open Questions
- 사과를 먹은 후 짧은 파티클(반짝임)을 추가할지 — 일단 v0.3에서는 머리 펄스만, 파티클은 별도 버전 검토
