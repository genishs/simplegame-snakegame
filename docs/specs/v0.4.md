# v0.4 Spec

## Goal
첫 진입 플레이어를 위한 "튜토리얼 스테이지"를 도입한다. 5×5 격자, 매우 느린 속도, 사과 3개 클리어. 게임에 처음으로 "스테이지" 개념이 들어온다.

## User Stories
- 처음 게임을 켠 플레이어로서, 작고 안전한 공간에서 조작에 익숙해지고 싶다.
- 클리어 후, 본 게임(더 큰 보드)으로 자연스럽게 넘어가고 싶다.
- 어떤 단계에 있는지 화면에서 알고 싶다.

## Acceptance Criteria
- [ ] 게임 시작 시 보드가 5×5 논리 격자로 시작한다 (캔버스 픽셀 영역은 그대로 400×400)
- [ ] 5×5 영역은 캔버스 중앙에 배치되고, 그 바깥 영역은 옅게(배경의 어두운 톤) 처리된다 — 스포트라이트
- [ ] 튜토리얼에서 뱀은 길이 2로 시작 (3 → 5×5에선 너무 큼)
- [ ] 튜토리얼 틱 속도 = 200ms (일반 스테이지 110ms 대비 훨씬 느림)
- [ ] 사과 3개를 먹으면 튜토리얼이 클리어된다
- [ ] 클리어 즉시 "튜토리얼 클리어!" 메시지가 0.8초 표시되고, 자동으로 Stage 1 (20×20, 110ms)로 전환된다
- [ ] Stage 전환 시 점수는 누적된다 (튜토리얼에서 +30점이면 그대로 30점에서 시작)
- [ ] Stage 전환 시 뱀 위치/길이는 새 보드 기준으로 리셋된다 (튜토리얼 잔여 상태가 끌려오지 않는다)
- [ ] HUD에 "Stage: Tutorial" 또는 "Stage: 1" 라벨이 표시된다
- [ ] 게임 오버 시 다시 시작하면 튜토리얼부터 시작된다
- [ ] `VERSION` = `0.4`, 페이지 표시 = `v0.4`
- [ ] `HISTORY.md`에 v0.4 항목 추가
- [ ] 디자인 토큰: 스포트라이트 외곽 색을 `STYLE.md`에 추가

## Out of Scope
- Stage 2 이후 (난이도 변화는 별도 버전)
- 스테이지 선택 UI
- 스테이지 클리어 통계/뱃지
- 보드 확장 애니메이션 (즉시 전환)

## Non-functional notes
- 페이로드 예산 50KB 유지
- 빌드 도구 없음
- 키 입력 매핑 변경 없음
- 성능: 스테이지 전환은 `tick()` 안에서 동기 수행 (별도 setTimeout/Promise 없음)

## Open Questions
- 튜토리얼 진행 중 게임 오버 처리 — 일단 일반 게임 오버와 동일하게 처리하고 재시작 시 튜토리얼부터. 별도 "다시 도전" UI는 v0.5 이후.
- 5×5에서 사과/뱀 시각이 너무 커 보이지 않는지 — 디자인 파트가 검증.
