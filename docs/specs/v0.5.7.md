# v0.5.7 Spec

## Goal

UI 스케일링, 난이도 곡선 완화, 한국어 전면 적용, 꾸물거리는 소화 wiggle 모션, 도움말 화면 — 5개 피드백을 단일 마이너 릴리스로 통합하여 플레이어 첫인상을 전반적으로 개선한다.

## User Stories

- (UI 스케일링) 플레이어로서, 화면 크기에 맞게 게임 보드가 크게 표시되기를 원한다. 그래서 작은 화면에서도, 큰 모니터에서도 보드가 적절한 크기로 채워져 쾌적하게 플레이할 수 있다.
- (난이도 곡선) 처음 하는 플레이어로서, Stage 1·2·3이 충분히 느려서 익숙해질 시간이 있기를 원한다. 그래서 첫 진입에서 좌절하지 않고 게임 메커닉을 자연스럽게 배울 수 있다.
- (한글화) 한국어 사용자로서, HUD·오버레이·버튼·힌트 바·타이틀 모든 텍스트가 한국어로 표시되기를 원한다. 그래서 영문 혼재 없이 게임 상태와 조작 방법을 직관적으로 파악할 수 있다.
- (wiggle 디지스천) 플레이어로서, 사과를 먹으면 뱀 몸통 속 덩어리가 꾸물거리며 이동하는 것이 보이기를 원한다. 그래서 사과를 삼키고 소화하는 느낌이 생동감 있게 전달된다.
- (도움말 화면) 처음 게임을 여는 플레이어로서, 조작 방법과 클리어 조건을 한 눈에 확인할 수 있는 도움말 화면이 보이기를 원한다. 그래서 아무 설명 없이 화면 앞에서 당황하지 않아도 된다.

## Acceptance Criteria

### Set A — UI 스케일링

- [ ] 캔버스 크기가 viewport에 비례하여 결정된다. CSS `clamp()` 또는 `min()` + viewport 단위(`vw`, `vh`)를 사용한다.
- [ ] 데스크탑(landscape, min-width 1024px): 캔버스 표시 크기가 `min(85vh, 85vw, 560px)`로 결정되며, 가로/세로 동일한 정사각형을 유지한다. design-lead STYLE.md v0.5.7 섹션 결정에 따라 cozy 비율 유지 + wiggle/wobble 가시 범위 보존 위해 spec 초안 상한보다 타이트하게 조정.
- [ ] 태블릿(768px–1023px 미디어쿼리): 캔버스 표시 크기가 `min(85vh, 85vw, 480px)`로 결정된다. design-lead STYLE.md v0.5.7 섹션 결정에 따라 cozy 비율 유지 + wiggle/wobble 가시 범위 보존 위해 spec 초안 상한보다 타이트하게 조정.
- [ ] 모바일 세로(portrait, max-width 767px): 캔버스 표시 크기가 `min(94vw, max(100px, calc(94vh - var(--mobile-controls-reserve, 80px) - 16px)))`로 결정되고 하단 모바일 버튼 행이 캔버스 아래 배치되어 잘리지 않는다. design-lead STYLE.md v0.5.7 섹션 결정에 따라 cozy 비율 유지 + wiggle/wobble 가시 범위 보존 위해 spec 초안 상한보다 타이트하게 조정.
- [ ] 모바일 가로(landscape, max-height 500px): 데스크탑 fallback(`min(85vh, 85vw, 560px)`)을 유지하고 좌우 모바일 버튼이 캔버스 옆에 배치되어 잘리지 않는다.
- [ ] 캔버스 내부 그리드는 그대로 20×20을 유지한다. `cols`, `rows` 데이터 변경 없음.
- [ ] 캔버스 backing store(실제 픽셀 해상도)는 표시 크기와 독립적으로 devicePixelRatio에 맞게 설정된다 (기존 HiDPI 처리 유지).
- [ ] 셀 픽셀 크기(`cellSize`)는 `canvas.width / cols`로 동적 계산되어, 표시 크기가 커져도 그리드 논리 크기는 변하지 않는다.
- [ ] 스케일링 변경 후 뱀 드로잉, 사과 드로잉, 오버레이 DOM, 모바일 버튼 터치 영역이 모두 올바르게 동작한다.

### Set B — 난이도 곡선 시프트

- [ ] 튜토리얼 틱이 **400ms** 이상이다 (`STAGES[0].tick >= 400`). memory 규칙(feedback_tutorial_non_punishing: tick ≥ 320ms)을 준수한다.
- [ ] Stage 1 틱이 **230ms** 이하이다 (`STAGES[1].tick <= 230`). 현 140ms에서 완화된다.
- [ ] Stage 2 틱이 **190ms** 이하이다 (`STAGES[2].tick <= 190`).
- [ ] Stage 3 틱이 **160ms** 이하이다 (`STAGES[3].tick <= 160`).
- [ ] 스테이지 1·2·3은 메모리 규칙(project_stage_difficulty: "stages 1–3 must feel very easy")을 충족하는 속도다. 개발자가 실제 플레이 검증 후 상한 이하에서 최종값을 결정하며, 결정된 값을 `HISTORY.md`에 기재한다.
- [ ] `STAGES` 배열 리터럴 이외의 게임 루프 로직(`loop()`, `tickAccum`)은 변경하지 않는다.

### Set D — 꾸물거리는 소화 wiggle 모션

- [ ] v0.5.3에서 구현된 소화 덩어리(입→꼬리 lerp) 위에 wiggle 진폭이 추가된다. 덩어리의 이동 경로가 일직선이 아니라 굼틀거리며 진행한다.
- [ ] wiggle 구현 방식: 덩어리의 진행 방향에 수직(normal)인 오프셋을 `sin(t * freq)` 형태로 계산하여 렌더 위치에 더한다. 수직 방향은 덩어리가 현재 이동 중인 세그먼트의 방향 벡터에서 유도한다.
- [ ] wiggle 진폭은 cellSize의 20% 이하(`amplitude <= cellSize * 0.2`)로 제한하여 인접 셀을 벗어나지 않는다.
- [ ] wiggle 주파수는 덩어리 하나당 약 1.5–2.5 사이클/초 범위로, 시각적으로 부드럽고 귀여운 느낌을 준다.
- [ ] 소화 덩어리가 꼬리에 도달(흡수 완료) 하면 wiggle도 함께 소멸한다. 흡수 완료 시점은 v0.5.3 bulge 소멸 로직과 동일하다.
- [ ] 복수의 사과를 연속으로 먹었을 때 덩어리 각각이 독립적으로 wiggle한다 (phase 오프셋 달리하여 동기화 방지).
- [ ] wiggle이 뱀 머리 또는 몸통의 기존 애니메이션(eat squash, bulge)과 시각적으로 충돌하지 않는다.
- [ ] wiggle 연산은 기존 RAF 루프 내에서 수행되며 별도 타이머(`setInterval`) 사용 금지.
- [ ] design-lead가 수직/수평/합성 중 최종 방향을 플레이 테스트로 확정한다. 본 spec은 수직(normal) 오프셋을 기본 구현 방향으로 지정하되, design-lead의 피드백에 따라 진폭·주파수 상수를 조정하는 것을 허용한다.

### Set E — 도움말 화면

#### E-1. 진입 조건

- [ ] 게임 최초 로드 시 (`localStorage` 키 `snakegame.helpSeen`이 없거나 `"false"`일 때) 도움말 화면이 CHOICE 화면 진입 전에 자동으로 표시된다. 기존 `snake-best` 키와 컨벤션 분리, 다른 서비스 키와 충돌 방지 목적으로 네임스페이스를 포함한 키 이름 사용.
- [ ] 도움말을 닫으면 `localStorage.setItem("snakegame.helpSeen", "true")`가 설정된다. 이후 재방문 시 도움말이 자동으로 나타나지 않는다.
- [ ] 도움말 화면은 언제든지 다시 볼 수 있는 진입 경로가 있다. 위치: CHOICE 화면의 "도움말 보기" 링크 또는 버튼 (디자인 세부사항은 design-lead가 결정).

#### E-2. 도움말 내용 (카피 초안)

도움말 화면은 아래 세 섹션으로 구성된다.

**[조작 방법]**

```
← → 또는 A D   뱀을 왼쪽 / 오른쪽으로 돌려요
스페이스바       시작 · 일시정지 · 재시작
Esc             튜토리얼 건너뛰기
1 / 2           튜토리얼 시작 / 바로 게임 시작
```

모바일이에요? 화면 왼쪽 / 오른쪽 버튼으로 회전해요!

**[스테이지 1–5 클리어 방법]**

사과를 N개 먹으면 클리어! 다음 스테이지로 넘어가요. 🍎

**[더 멀리 가면…]**

스테이지 6부터는 보드를 가득 채워야 클리어예요. 뱀이 온 보드를 뒤덮는 날을 기대해주세요! (곧 업데이트)

- [ ] 도움말 화면에 위 세 섹션이 모두 포함된다.
- [ ] 조작 방법 섹션에 키보드 키와 동작 설명이 표 또는 목록 형태로 표시된다.
- [ ] 스테이지 1–5 클리어 조건이 "사과 N개" 형태로 표시된다. N은 현재 `STAGES[1–5].clearAfterApples` 값을 참조하며, 스테이지마다 다를 경우 스테이지별로 나열한다.
- [ ] v0.6 풀맵 클리어 예고 문구가 포함된다. 단, 실제 메커닉은 구현하지 않는다.
- [ ] 도움말 화면에 "닫기" 버튼이 있으며 클릭/탭/Space/Esc로 닫을 수 있다.
- [ ] 모바일 환경에서 도움말 텍스트가 잘리지 않고 스크롤 가능하다.
- [ ] 도움말이 열려 있는 동안 게임은 시작되지 않는다 (STATE는 READY 또는 CHOICE 이전 단계에서 대기).

#### E-3. 접근성

- [ ] 도움말 컨테이너에 `role="dialog"` 및 `aria-modal="true"` 속성이 있다.
- [ ] 도움말이 열리면 "닫기" 버튼에 포커스가 이동한다.
- [ ] 도움말이 닫히면 포커스가 이전 요소(CHOICE 버튼 또는 캔버스)로 복귀한다.

- (미해소) 도움말 화면 스테이지별 사과 N개 표기: Stage 1–5 모두 동일한 `clearAfterApples`인지 스테이지별로 다른지 개발자가 현재 데이터 확인 후 카피 최종 확정.

## Non-functional Notes

### 페이로드 예산

- 전체 목표: 50KB 미만 유지.
- 한글화 순증가: < 1KB (ASCII → UTF-8 치환, 대상 문자열 총량 수백 바이트).
- wiggle 로직 추가: 예상 < 0.5KB (sin 계산 + phase 오프셋, 새 라이브러리 없음).
- 도움말 HTML/JS 추가: 예상 < 1.5KB.
- CSS 스케일링 변경: < 0.3KB.
- 전체 순증가 예상: < 3.5KB → 기존 ~32KB에서 ~35.5KB 예상, 상한 이내.

### 접근성

- `lang="ko"` 속성이 `<html>` 태그에 이미 설정되어 있어 스크린리더 언어 프로파일 자동 전환.
- aria-label 한글화 후 iOS VoiceOver / Android TalkBack 한국어 환경에서 버튼 기능이 명확히 전달된다.
- 도움말 dialog는 `role="dialog"` + `aria-modal="true"` + 포커스 트랩으로 접근성 확보.

### 성능

- wiggle은 기존 RAF 루프 내 수학 연산(`Math.sin`)만 추가. GPU 드로우콜 수 변경 없음.
- 스케일링 CSS 변경은 레이아웃 재계산 1회(로드 시 + 리사이즈 이벤트)에만 발생.
- 도움말 화면은 DOM 오버레이 기반. 캔버스 draw 없음.

### 모바일

- 스케일링 변경으로 모바일 터치 영역이 확장된다. 기존 44px 최소 터치 영역 규칙을 유지한다.
- 모바일 세로/가로 모드 각각에서 보드와 버튼이 잘리지 않음을 DevTools 에뮬레이터 및 실기기로 검증한다.
- 도움말 텍스트는 `overflow-y: auto` 처리하여 소형 화면에서도 스크롤 가능하다.

### 한글 카피 톤 가이드

- 친근하고 아기자기한 말투 ("먹으면 클리어!", "기대해주세요!")
- 명사형 종결 (`시작/일시정지/재시작`) 과 동사형 종결 ("돌려요", "넘어가요") 을 문맥에 따라 병행 사용.
- 느낌표는 긍정적 문장에만, 부정/경고 문장에는 사용하지 않음 ("게임 끝" — 느낌표 없음).
- Space 키: 힌트 바·오버레이에서 "스페이스"; README Controls 표 Key 열에서는 국제 표준 `Space` 유지.

### wiggle 진폭·주파수 기준값 (design-lead 협의용)

| 파라미터 | 기준값 | 허용 범위 | 비고 |
|---|---|---|---|
| amplitude | `cellSize * 0.15` | `0.05–0.20` | 인접 셀 침범 방지 |
| frequency | `2.0` (사이클/초) | `1.5–2.5` | 귀엽고 부드러운 느낌 |
| phase offset (덩어리 간) | `Math.PI / 3` | 자유 | 동기화 방지 |

### 난이도 곡선 검증 기준

- 튜토리얼: 키보드를 처음 쓰는 성인이 4회 연속 죽지 않고 사과 3개를 먹어야 한다.
- Stage 1: 튜토리얼을 통과한 플레이어가 1회 플레이에서 벽 충돌 없이 5개를 먹을 확률 > 70%.
- Stage 2·3: 명확한 체감 속도 차이가 있어야 한다.

## Decisions

- **(A) 5개 피드백을 단일 v0.5.7로 통합** — 사용자가 v0.5.7→v0.5.8→v0.5.9 분할 계획을 명시적으로 철회하고 단일 마이너 통합을 결정했다. 5개 항목 모두 새 게임 메커닉(풀맵 클리어 등)이 아닌 기존 메커닉의 품질 개선이므로 메이저 번호(v0.6)를 올리지 않는다. 동시에 변경 범위가 충분히 크므로 패치(v0.5.6.x)가 아닌 마이너(v0.5.7)로 처리한다.

- **(B) 도움말 첫 진입 감지: localStorage 방식 채택** — 세 가지 후보를 검토했다. ① 항상 노출 후 닫기: 재방문 시마다 도움말이 열려 숙련 플레이어에게 방해가 된다. ② 매 세션 1회: sessionStorage 기반이면 탭을 닫고 다시 열 때마다 표시되어 재방문 사용자에게 과도하다. ③ localStorage 최초 1회 + 재열람 경로 제공: 첫 방문자에게 자동 표시하고 이후 도움말이 필요한 경우 직접 열 수 있게 하여 두 그룹을 모두 만족시킨다. localStorage 지원이 없는 환경(Private 모드 등)에서는 항상 도움말이 표시되는 fallback을 제공한다.

- **(C) wiggle 방향: 수직(normal) 오프셋 기본 채택** — 수직 진동은 뱀의 이동 방향과 무관하게 "굼틀거리는" 생물 느낌을 주며, 직선 구간에서 진폭이 가장 잘 보인다. 수평 진동(이동 방향과 동일 축)은 속도 변화로 오해될 수 있다. 합성(수직+수평)은 구현이 복잡하고 진폭이 작은 셀 범위에서는 차이가 미미하다. design-lead가 플레이 테스트 후 합성을 요청하면 진폭 파라미터 추가로 대응한다.

- **(D) 난이도 곡선: 상한값 사전 결정 후 개발자 검증** — 스펙 단계에서 정확한 "최적값"을 확정하면 개발자 플레이 경험을 무시하게 된다. 상한(튜토리얼 400ms / Stage 1 230ms / Stage 2 190ms / Stage 3 160ms)을 지키는 범위에서 개발자가 플레이 테스트 후 최종값을 결정하고, 그 값과 근거를 `HISTORY.md`에 기재한다. memory 규칙(project_stage_difficulty, feedback_tutorial_non_punishing) 준수 여부는 SCM 리뷰 단계에서 확인한다.

- **(E) i18n 구조 미도입** — 현재 단계에서 다국어 지원 계획이 없다. locale 파일/namespace/동적 로딩 구조는 50KB 페이로드 예산 및 단일 파일 아키텍처와 맞지 않는다. 향후 i18n이 필요해지는 시점에 요구사항이 더 구체화될 것이므로, 지금은 단순 문자열 치환으로 충분한 효과를 얻는다.

- **(F) v0.6은 풀맵 클리어 메커닉으로 유지** — 도움말 화면에서 v0.6 예고를 포함하지만 메커닉을 구현하지 않는다. 예고 문구는 플레이어의 장기 기대감을 형성하고, 개발팀이 v0.6 방향을 공개적으로 확약하는 역할을 한다.
