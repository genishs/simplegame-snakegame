# v0.5.6 Spec

## Goal

게임 시작 직전에 "튜토리얼 시작 / 건너뛰기" 선택 화면(STATE.CHOICE)과 3-2-1 카운트다운 화면(STATE.COUNTDOWN)을 끼워, 신규/재시작 시 사용자가 튜토리얼 진입 여부를 매번 결정하고 차분히 준비한 뒤 시작할 수 있게 한다.

## User Stories

- 플레이어로서, 게임을 처음 시작하거나 재시작할 때 튜토리얼 여부를 직접 선택하고 싶다. 그래서 원하지 않는 튜토리얼을 강제로 겪지 않아도 되고, 처음 하는 사람은 안내를 받을 수 있다.
- 플레이어로서, 선택 후 3초 카운트다운이 시작되면 차분히 손을 준비하고 싶다. 그래서 "시작" 버튼을 눌렀더니 갑자기 뱀이 움직이는 상황을 피할 수 있다.
- 플레이어로서, 카운트다운 중 기다리기 싫을 때 Space 또는 Esc로 즉시 시작하고 싶다. 그래서 3초를 억지로 기다리지 않아도 된다.
- 이미 게임 중이던 플레이어로서, 일시정지 후 재개할 때 선택 화면이나 카운트다운 없이 바로 이어서 하고 싶다. 그래서 흐름이 끊기지 않는다.
- 스테이지를 클리어한 플레이어로서, 다음 스테이지로 자동으로 넘어갈 때 선택 화면이나 카운트다운 없이 진행하고 싶다. 그래서 스테이지 전환이 부드럽게 이어진다.

## Acceptance Criteria

### Set A — STATE 추가

- [ ] `STATE.CHOICE`와 `STATE.COUNTDOWN` 두 개의 새 상태가 `STATE` 객체에 추가된다.
- [ ] 기존 `STATE.READY`, `STATE.PLAYING`, `STATE.PAUSED`, `STATE.BLOCKED`, `STATE.STAGE_CLEAR`, `STATE.OVER`는 값, 의미, 전환 조건 모두 변경되지 않는다.

### Set B — CHOICE 화면

- [ ] READY 또는 OVER 상태에서 `auxAction()` 호출 시 (Space 키, 캔버스 탭, aux 버튼, 마우스 클릭 모두) `STATE.CHOICE`로 진입한다.
- [ ] CHOICE 오버레이 DOM에는 타이틀 "천천히 시작해볼까요?", 서브타이틀 "처음이라면 튜토리얼을 추천해요", 두 버튼 "튜토리얼부터 시작"과 "바로 게임 시작"이 표시된다.
- [ ] 키보드 `1` 키 → "튜토리얼부터 시작" 선택 → COUNTDOWN 진입.
- [ ] 키보드 `2` 키 → "바로 게임 시작" 선택 → COUNTDOWN 진입.
- [ ] 키보드 `←` 또는 `→` 키 → CHOICE 화면에서 두 버튼 간 강조 이동 (포커스 이동 및 시각 강조). 해당 키로 `applyTurn`은 호출되지 않는다.
- [ ] 키보드 `Space` 키 → 현재 강조된 버튼 확정 → COUNTDOWN 진입.
- [ ] CHOICE 화면 최초 진입 시 "튜토리얼부터 시작" 버튼이 기본 강조 상태이다 (신규 사용자 친화). 게임오버 후 재진입 시에도 동일하게 항상 "튜토리얼부터 시작"이 기본 강조이다 (마지막 선택 기억 없음).
- [ ] 모바일에서 버튼 직접 탭 → 해당 선택 → COUNTDOWN 진입.
- [ ] 마우스로 버튼 클릭 → 해당 선택 → COUNTDOWN 진입.
- [ ] 캔버스 `pointerdown`이 CHOICE 상태에서 발생하면 no-op (버튼 직접 탭/클릭만 인식, 캔버스 탭은 무시).
- [ ] CHOICE 상태에서 회전 키(`←`, `→`, `A`, `D`)는 강조 이동 이외의 회전 동작(`applyTurn`)을 실행하지 않는다. keydown 핸들러가 CHOICE 상태일 때 `←`/`→`는 강조 이동 분기로 처리한다.

### Set C — COUNTDOWN 화면

- [ ] CHOICE에서 선택이 확정되는 즉시 `STATE.COUNTDOWN`으로 진입한다.
- [ ] 카운트다운 진행 중 보드 위에 `--text-strong` (#3b2a1a) @ 0.35 alpha의 마스크가 그려진다 (별도 토큰; `--mask-outside`와 혼용 금지).
- [ ] 마스크 위 캔버스 중앙에 숫자 3 → 2 → 1 순서로 각각 1초씩 표시된다. 각 숫자의 타이밍: 180ms 페이드 인 + 640ms 홀드 + 180ms 페이드 아웃 (합계 1,000ms).
- [ ] 페이드 인: scale 0.7→1.0 + alpha 0→1 동시 변화.
- [ ] 페이드 아웃: alpha 1→0만 변화 (scale 고정). v0.5.3 bulge 페이드 원칙과 동일.
- [ ] 1이 페이드 아웃 완료(3초 경과) 후 즉시 `STATE.PLAYING`으로 전환한다.
- [ ] 숫자 폰트는 `Segoe UI Rounded, SF Pro Rounded, Apple SD Gothic Neo, sans-serif` 순 폴백, 120px, weight 700, 색상 `--text-strong`.
- [ ] 캔버스 하단 소형 안내문 "Space · Esc — 바로 시작"이 카운트다운 중 표시된다.
- [ ] `Space` 또는 `Esc` 키 입력 → 카운트다운 즉시 종료 → `STATE.PLAYING` 진입.
- [ ] 캔버스 `pointerdown`이 COUNTDOWN 상태에서 발생하면 카운트다운을 즉시 스킵하고 PLAYING으로 진입한다.
- [ ] COUNTDOWN 중 회전 키(`←`, `→`, `A`, `D`)는 무시한다.
- [ ] COUNTDOWN 중 `Esc` 키는 카운트다운 스킵으로만 동작하며, 튜토리얼 진행 중 Esc skip 동작(`advanceStage`)은 COUNTDOWN 상태에서 실행되지 않는다.
- [ ] COUNTDOWN 중 일시정지는 불가능하다. Space는 스킵, Esc는 스킵으로만 작동한다.

### Set D — PLAYING 진입 및 초기화

- [ ] CHOICE에서 "튜토리얼부터 시작" 선택 → COUNTDOWN 종료 후 `loadStage(0)` (Tutorial) 호출 → PLAYING 진입.
- [ ] CHOICE에서 "바로 게임 시작" 선택 → COUNTDOWN 종료 후 `loadStage(1)` (Stage 1) 호출 → PLAYING 진입.
- [ ] PLAYING 진입 시 점수는 0으로 리셋된다.
- [ ] 뱀 위치, 사과 위치, 뱀 길이는 `loadStage` 호출로 초기화된다 (별도 처리 없음).

### Set E — 우회 흐름 (회귀)

- [ ] `STATE.PAUSED` 상태에서 Space 키 / aux 버튼 탭 / 캔버스 탭 → 즉시 `STATE.PLAYING` (CHOICE, COUNTDOWN 우회).
- [ ] `STATE.STAGE_CLEAR` 상태에서 800ms 자동 홀드 후 다음 스테이지로 자동 전환 → 즉시 `STATE.PLAYING` (CHOICE, COUNTDOWN 우회).
- [ ] `STATE.BLOCKED` 상태에서 회전 입력 → `tryUnblock` 경유 (CHOICE, COUNTDOWN 무관).
- [ ] 튜토리얼 진행 중 `Esc` 키 → `advanceStage` 호출 (CHOICE, COUNTDOWN 무관).
- [ ] OVER 오버레이가 닫히고 CHOICE 오버레이가 열릴 때 페이드 없이 단순 교체된다.

### Set F — 캔버스 pointerdown 분기 (폴백 통합)

- [ ] 캔버스 `pointerdown` 핸들러가 `STATE.READY` 또는 `STATE.OVER` 상태일 때 → `auxAction()` 호출 → CHOICE 진입 (v0.5.5 폴백 동작 유지).
- [ ] 캔버스 `pointerdown` 핸들러가 `STATE.PAUSED` 상태일 때 → `auxAction()` 호출 → 즉시 PLAYING (CHOICE 우회, v0.5.5 동작 유지).
- [ ] 캔버스 `pointerdown` 핸들러가 `STATE.CHOICE` 상태일 때 → no-op.
- [ ] 캔버스 `pointerdown` 핸들러가 `STATE.COUNTDOWN` 상태일 때 → 카운트다운 즉시 스킵 → PLAYING 진입.
- [ ] 캔버스 `pointerdown` 핸들러가 `STATE.PLAYING` 또는 `STATE.BLOCKED` 상태일 때 → 기존 좌/우 회전 동작 유지 (변경 없음).

### Set G — keydown 핸들러 변경

- [ ] `1` 키: CHOICE 상태에서 "튜토리얼부터 시작" 선택. 그 외 상태에서는 기존 동작.
- [ ] `2` 키: CHOICE 상태에서 "바로 게임 시작" 선택. 그 외 상태에서는 기존 동작.
- [ ] `Space` 키: CHOICE 상태에서 현재 강조된 버튼 확정 / COUNTDOWN 상태에서 즉시 스킵 / 그 외 상태에서 기존 start·pause·resume 동작.
- [ ] `Esc` 키: COUNTDOWN 상태에서 즉시 스킵 / 그 외 상태에서 기존 튜토리얼 skip(`advanceStage`) 동작.
- [ ] `←`, `→`, `A`, `D` 키: CHOICE 상태에서 강조 이동 / COUNTDOWN 상태에서 무시 / 그 외 상태에서 기존 `applyTurn` 동작.

### Set H — 메타파일

- [ ] `VERSION` 파일이 `0.5.6`으로 갱신된다.
- [ ] `index.html` 버전 표기가 `v0.5.6`으로 갱신된다.
- [ ] `HISTORY.md` 최상단에 v0.5.6 항목이 추가된다. STATE.CHOICE / STATE.COUNTDOWN 추가, 입력 분기 변경, PAUSED·STAGE_CLEAR 우회 결정, edge case 처리 근거 포함.

## Open Questions

- (해소됨) CHOICE 진입 시 기본 강조 버튼: 항상 "튜토리얼부터 시작" 강조. 마지막 선택 기억(localStorage) 없음. 신규 사용자 친화를 우선한다.
- (해소됨) 게임오버 직후 CHOICE 재진입 시 기본 강조: 항상 "튜토리얼부터 시작" 강조 (단순화).
- (해소됨) COUNTDOWN 중 일시정지 가능 여부: 불가능. Space → 스킵, Esc → 스킵. 카운트다운은 스킵하거나 기다리는 두 가지 선택만 있다.
- (해소됨) CHOICE/COUNTDOWN 중 페이지 visibility 변경(탭 전환): 별도 처리 없음. RAF 자체가 백그라운드에서 멈추므로 카운트다운 타이머도 자연히 중단된다.
- (해소됨) OVER → 캔버스 탭 → CHOICE 시 시각 전환: OVER 오버레이 닫힘 + CHOICE 오버레이 열림, 페이드 없이 단순 교체.
- (해소됨) CHOICE 캔버스 탭: no-op. 버튼 클릭 위치 판별이 캔버스 좌표로는 어렵고 DOM 버튼이 존재하므로 버튼 직접 탭/클릭만 인식한다.

## Non-functional Notes

### 페이로드

- 예상 순증가량 +2.9KB (기존 ~29.3KB → ~32.2KB, 50KB 상한 이내).

### 성능

- 카운트다운 draw: 마스크 fillRect 1회 + 숫자 fillText 1회 + 안내문 fillText 1회 = 3 draw calls/frame. 60fps 안전 기준 이내.
- CHOICE 화면은 DOM 오버레이 기반 (canvas draw 없음). 성능 영향 없음.
- RAF 기반 타이머로 카운트다운 진행 → `setInterval` 사용 금지 (탭 전환 시 RAF가 멈추면 자연히 중단).

### 접근성

- CHOICE 버튼은 `<button>` 요소로 구현하여 스크린리더 및 키보드 포커스 접근성 확보.
- 키보드 `1`/`2`/`Space`/`←`/`→` 모두 CHOICE 선택 가능.
- 카운트다운 중 안내문 "Space · Esc — 바로 시작"은 시각적으로 노출.

### 모바일 고려사항

- CHOICE 버튼 두 개는 각각 44px 이상 터치 영역 확보.
- COUNTDOWN 중 캔버스 탭으로 스킵 가능하므로 모바일에서도 즉시 시작 접근성 유지.
- 모바일 하단 aux 버튼이 READY/OVER에서 `auxAction()` 호출 → CHOICE 진입 (v0.5.5 폴백 그대로).

## Decisions

- **(A) CHOICE 진입 조건을 auxAction()으로 통합** — READY/OVER에서 Space 키, 캔버스 탭, aux 버튼, 마우스 클릭 모두 `auxAction()`을 경유하여 CHOICE로 진입한다. 기존 `auxAction` 호출 경로를 재사용하여 분기 추가 최소화.
- **(B) PAUSED/STAGE_CLEAR 우회** — 재개 및 다음 스테이지 전환은 사용자가 이미 게임 맥락 안에 있으므로 선택 화면이 불필요하다. 흐름 끊김 방지를 위해 CHOICE/COUNTDOWN을 우회한다.
- **(C) CHOICE 기본 강조: 항상 튜토리얼** — 마지막 선택 기억(localStorage)은 구현 복잡도를 높이고 v0.5.6 범위를 벗어난다. 신규 사용자 친화를 우선하여 항상 "튜토리얼부터 시작"을 기본 강조로 고정한다.
- **(D) COUNTDOWN 중 일시정지 금지** — 카운트다운 구간은 게임 시작 직전 준비 단계이다. 이 구간에서 일시정지 개념을 도입하면 상태 전환 그래프가 불필요하게 복잡해진다. 스킵(Space/Esc/캔버스 탭)으로 충분히 탈출 가능하다.
- **(E) CHOICE 캔버스 탭 = no-op** — 캔버스 좌표만으로 어느 버튼을 탭했는지 판별하기 어렵고, DOM `<button>` 요소가 이미 존재한다. 캔버스 탭은 no-op으로 처리하고 버튼 직접 탭/클릭으로만 선택한다.
- **(F) RAF 기반 카운트다운 타이머** — `setInterval`은 백그라운드 탭에서 throttle되어 의도치 않게 빨리 진행될 수 있다. `requestAnimationFrame` + `performance.now()` delta로 진행하면 탭 전환 시 RAF가 멈춰 자연스럽게 일시 중단된다.
- **(G) 카운트다운 마스크 토큰 별도 정의** — `--mask-outside`(rgba(120,90,60,0.18))는 튜토리얼 spotlight용이다. 카운트다운 마스크는 더 진한 `--text-strong` 기반 0.35 alpha이며 용도가 다르므로 별도 토큰으로 정의한다.
