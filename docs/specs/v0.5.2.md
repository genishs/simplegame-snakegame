# v0.5.2 Spec

## Goal

뱀을 이어진 단일 곡선 몸체 + 계란형 머리로 재드로잉하여 시각적으로 살아있는 캐릭터처럼 보이게 하고, 먹기 모션에 squash & stretch를 추가한다. 게임 로직과 격자 좌표계는 변경하지 않는다.

## User Stories

- 플레이어로서, 뱀의 몸이 격자 칸 경계 없이 부드럽게 이어지는 것을 보고 싶다. 그래서 캐릭터처럼 느껴지는 뱀을 즐길 수 있다.
- 플레이어로서, 뱀의 머리가 진행 방향에 따라 눈·혀가 달린 계란형 실루엣으로 보이고 싶다. 그래서 뱀이 어느 쪽을 향하는지 직관적으로 파악할 수 있다.
- 플레이어로서, 회전 구간에서 뱀이 직각으로 꺾이지 않고 부드럽게 휘는 것을 보고 싶다. 그래서 움직임이 자연스럽게 느껴진다.
- 플레이어로서, 사과를 먹을 때 머리가 squash & stretch로 튕기는 것을 보고 싶다. 그래서 먹기 동작이 명확하게 전달된다.

## Acceptance Criteria

### Set A — 머리 실루엣

- [ ] 머리는 `ctx.ellipse()`로 그려지며, facing 축 방향 반지름 = `CELL × 0.55` (diameter ≈ `CELL × 1.10`), 수직 축 반지름 = `CELL × 0.46` (diameter ≈ `CELL × 0.92`)이다 (`--head-length = CELL × 1.10`, `--head-width = CELL × 0.92`).
- [ ] 머리 채우기 색은 `--snake-head` (`#6bb96b`)이다.
- [ ] 눈 흰자 반지름은 `2.5px` (`--head-eye-radius`), 동공 반지름은 `1.2px` (`--head-pupil-radius`), 동공 색은 `#2a2018` (`--head-pupil-color`)이다.
- [ ] 눈은 머리 중심에서 진행 방향으로 `CELL × 0.18` 앞, 중심선 양옆 `CELL × 0.22` 간격에 배치된다 (`--head-eye-offset-forward`, `--head-eye-offset-side`).
- [ ] 혀끝은 머리 앞쪽 끝에서 진행 방향으로 `3px` 뻗으며 (`--head-tongue-length`), 색은 `#ef9aa6` (`--head-tongue-color`)이다. 혀는 `1600ms`마다 `120ms` 동안 표시되는 idle flicker로 동작한다 (`--head-tongue-visibility`).
- [ ] 머리 ellipse는 진행 방향 벡터로 회전하여 상·하·좌·우 4방향 실루엣이 모두 계란형으로 구분된다 (`--head-rotation = direction vector`).
- [ ] **캡처 요구 (PR 필수):** 머리 4방향(우·좌·상·하) 각 1장, 총 4장 — 각 방향에서 계란형 silhouette이 확인되어야 한다.

### Set B — 몸 연결 (capsule stroke)

- [ ] 몸 전체가 하나의 `beginPath … stroke()` 호출로 그려진다. segment마다 개별 `fill()`을 사용하지 않는다.
- [ ] stroke 색은 `--snake-body` (`#7cc47c`), `lineWidth = CELL × 0.86` (`--body-thickness`), `lineCap = round`, `lineJoin = round` (`--body-linecap`, `--body-linejoin`)이다.
- [ ] 몸 그림자는 동일 polyline을 `lineWidth = CELL × 0.86 + 2px`, 색 `--snake-shadow` (`rgba(60, 100, 60, 0.18)`)으로 먼저 stroke하여 구현한다 (`--body-shadow`).
- [ ] body polyline은 각 interior segment center에서 `quadraticCurveTo`를 사용한다 — 직전 edge 중점에서 시작, 다음 edge 중점으로 끝나며 segment center가 control point이다 (`--body-smoothing`).
- [ ] body polyline은 head 바로 뒤 segment부터 tail까지 이어지며, head ellipse와 자연스럽게 접합된다 (head-adjacent join은 head ellipse 아래로 가려짐, `--corner-head-handling`).
- [ ] 일직선 구간에서 segment 경계가 육안으로 식별되지 않는다.
- [ ] **캡처 요구 (PR 필수):** 일직선 5-segment 상태 1장 — segment 경계선이 보이지 않아야 한다.

### Set C — 회전 곡선

- [ ] 인접 segment의 방향이 바뀌는 cell (`prev dir != next dir`)은 corner cell로 판정되며, `quadraticCurveTo(cellCenter, edgeMidpointOut)`로 quarter-arc를 그린다. 호의 유효 반지름은 `CELL × 0.50` (`--corner-radius`, `--corner-detection`, `--corner-implementation`).
- [ ] tail 바로 앞 cell(꼬리 직전 segment)은 corner cell이더라도 라운딩을 적용하지 않고 직선 `lineTo`로 처리한다 (`--corner-tail-handling` — pinch 방지).
- [ ] corner 판정 로직은 render pass와 동일한 O(N) 단일 순회에서 수행되며, frame마다 별도 할당 없이 재사용 배열로 동작한다.
- [ ] **캡처 요구 (PR 필수):** ㄱ자 회전 1장, ㄴ자 회전 1장, U자 회전 1장 — 각 회전 cell에서 직각 꺾임 없이 곡선이 확인되어야 한다.

### Set D — 먹기 모션 (squash & stretch)

- [ ] 사과 접촉 시점부터 `180ms` 동안 머리에 squash & stretch가 적용된다 (`--eat-squash-dur`).
- [ ] squash 피크: facing 방향 scale = `1.18`, 수직 방향 scale = `0.88` (`--eat-squash-x`, `--eat-squash-y`).
- [ ] squash & stretch 이징은 ease-out(lunge) → ease-in(settle) 형태이며, 기존 `--eat-pulse-*` scale (`1.10`, `150ms`)과 `ctx.scale()` 수준에서 곱셈으로 합성된다.
- [ ] sparkle 토큰(`--sparkle-color`, `--sparkle-count`, `--sparkle-radius`, `--sparkle-dur`)은 STYLE.md에 정의되어 있으나 v0.5.2에서는 활성화하지 않는다.
- [ ] **캡처 요구 (PR 필수):** 먹기 직전 프레임, 먹기 직후 프레임(squash 피크), 180ms 경과 후 복귀 프레임, 총 3장 — squash 변형이 육안으로 확인되어야 한다.

### 드로잉 구현 위치 (game.js)

- [ ] `drawSnakeHead()` 함수가 ellipse + 눈 + 혀 + squash 합성 transform으로 교체된다.
- [ ] `drawSegment()` 함수가 제거되고, 동등한 body polyline 드로잉 함수(`drawSnakeBody()` 또는 동일 역할)로 대체된다.
- [ ] `draw()` 내 루프 `for (let i = snake.length - 1; i >= 1; i--) drawSegment(...)` 가 새 body 드로잉 호출로 교체된다.

## Out of Scope

- 소화 애니메이션 (food traveling along the body) — v0.5.3으로 명시적으로 분리
- sparkle 활성화 — 토큰만 예약, v0.5.2에서는 OFF
- 게임 로직·격자 좌표계·충돌 판정 변경
- 새 팔레트 토큰, HUD 레이아웃, 스테이지 데이터 변경
- per-length body gradient — v0.6+ 예정
- Canvas 외부 이미지·SVG DOM 사용
- 배경 패턴·애프터이미지·사망 시 표정 변경

## Non-functional Notes

### 페이로드
- 전체 페이로드 50KB 이하를 유지한다. Canvas 직접 드로잉만 사용하므로 추가 에셋 없음. 신규 draw 로직이 추가하는 코드는 수 KB 이내로 예상된다.

### 성능 (60fps)
- body polyline: `beginPath` 1회 + 최대 ~60 `lineTo`/`quadraticCurveTo` + `stroke` 1회 — 길이 60 기준 ~60 path commands, 1ms 예산 안전 (Design-Lead 확인, STYLE.md Performance budget 참조).
- head: `ellipse` 1회 + eye circle 4회 + tongue path (조건부) 1회 = 프레임당 최대 6 draw calls, 상수.
- corner detection: render pass와 동일한 O(N) 순회, frame마다 별도 배열 할당 없음.
- eat motion: pure `ctx.scale()` transform, 추가 draw call 없음.
- 정량 검증: 브라우저 DevTools Performance 패널에서 60-segment 길이 도달 시 frame time이 16.7ms를 초과하지 않음을 PR에서 확인한다.

### 접근성 / 모바일
- Canvas 드로잉이므로 키보드/WASD 조작은 기존 그대로 유지된다. 터치 조작은 이번 버전 범위 밖이다.

## Decisions

- **(A) capsule stroke 채택 (Set B)** — filled outline path 방식은 양쪽 경계 좌표 계산과 self-intersection 처리가 필요하고 꺾임부에서 fragile하다. capsule stroke(`lineJoin=round`)는 N개 점만으로 동일 결과를 더 안정적으로 달성하며 O(N) 1회 stroke로 충분하다 (STYLE.md Set B 참조).
- **(B) quadraticCurveTo Catmull-Rom-lite 채택 (Set B/C)** — 각 interior segment center를 control point로, 인접 edge 중점을 anchor로 하는 quadratic curve는 trig 없이 quarter-arc 수준의 곡률을 제공한다. 꼬리 직전 cell의 라운딩 OFF는 프로토타이핑에서 pinch가 확인된 데 따른 예외이다 (STYLE.md Set C `--corner-tail-handling`).
- **(C) squash & stretch를 eat-pulse와 곱셈 합성** — squash(`1.18 × 0.88`)와 pulse(`1.10`)는 head transform 위에 `ctx.scale()`로 순차 적용하여 분리 없이 단일 head draw call에서 처리된다. sparkle은 토큰만 예약하고 이번 버전에서는 OFF한다 (cozy 규칙 준수, STYLE.md Set D).
- **(D) head ellipse가 body polyline join을 가림** — head-adjacent corner의 특별 처리가 불필요해진다. body polyline은 항상 head 바로 뒤 segment부터 시작하며, head ellipse를 위에 그려 join을 덮는다 (STYLE.md Set C `--corner-head-handling`).
